# C1 职位类型分类方法详解
## Job Type Classification Methodology

---

## 🎯 分类方法概述

**方法**: 基于关键词匹配的规则分类法 (Keyword-based Rule Classification)

**原理**:
- 将30,000个不同的Job Title转换为小写
- 按优先级顺序匹配关键词
- 分配到10个标准化职位类型

---

## 📋 10个职位类型及其分类规则

### 1. **Management** (管理类)

**关键词**:
```python
['manager', 'director', 'executive', 'head', 'chief', 'president']
```

**典型职位示例**:
- Project Manager (项目经理)
- Marketing Director (市场总监)
- Chief Technology Officer (首席技术官)
- Executive Vice President (执行副总裁)
- Department Head (部门主管)

**为什么优先级最高**:
- "manager"等词通常最能体现职位本质
- 例如: "Senior Software Engineer Manager" → 虽然有"Engineer",但"Manager"更准确

---

### 2. **Engineering** (工程技术类)

**关键词**:
```python
['engineer', 'developer', 'programmer', 'architect', 'scientist']
```

**典型职位示例**:
- Software Engineer (软件工程师)
- Full Stack Developer (全栈开发者)
- Data Scientist (数据科学家)
- Solutions Architect (解决方案架构师)
- Systems Programmer (系统程序员)

**注意**:
- 包括传统工程(mechanical engineer, civil engineer)
- 也包括IT工程(software engineer, devops engineer)
- "scientist"也归入此类(research scientist, data scientist)

---

### 3. **Analysis** (分析咨询类)

**关键词**:
```python
['analyst', 'consultant', 'advisor', 'strategist']
```

**典型职位示例**:
- Business Analyst (商业分析师)
- Financial Consultant (财务顾问)
- Investment Analyst (投资分析师)
- Strategy Advisor (战略顾问)
- Research Strategist (研究策略师)

**为什么与Engineering分开**:
- Engineer侧重"构建/开发"
- Analyst侧重"分析/建议"
- 例如: "Data Engineer"(构建数据管道) vs "Data Analyst"(分析数据洞察)

---

### 4. **Medical_Professional** (医疗专业类)

**关键词**:
```python
['doctor', 'physician', 'nurse', 'surgeon', 'therapist', 'dentist']
```

**典型职位示例**:
- Family Physician (家庭医生)
- Registered Nurse (注册护士)
- Physical Therapist (理疗师)
- Orthopedic Surgeon (骨科医生)
- Dentist (牙医)

**覆盖范围**:
- 需要专业医疗执照的职位
- 在所有行业都可能出现(不仅限Healthcare)
- 例如: IT行业的"Nurse, children's" (数据中真实存在)

---

### 5. **Education** (教育培训类)

**关键词**:
```python
['teacher', 'professor', 'instructor', 'educator', 'trainer']
```

**典型职位示例**:
- High School Teacher (高中教师)
- University Professor (大学教授)
- Corporate Trainer (企业培训师)
- Technical Instructor (技术讲师)
- Online Educator (在线教育者)

**跨行业存在**:
- Education行业: 学校教师
- IT行业: 技术培训师
- Finance行业: 投资培训讲师
- Manufacturing行业: 工艺培训师

---

### 6. **Creative** (创意设计类)

**关键词**:
```python
['designer', 'artist', 'creative', 'photographer', 'writer']
```

**典型职位示例**:
- Graphic Designer (平面设计师)
- UX/UI Designer (用户体验设计师)
- Content Writer (内容撰稿人)
- Photographer (摄影师)
- Creative Director (创意总监)

**注意**:
- "Creative Director"会被归入**Management**而非Creative
- 因为"director"在关键词匹配中优先级更高

---

### 7. **Sales_Marketing** (销售市场类)

**关键词**:
```python
['sales', 'marketing', 'account', 'business development']
```

**典型职位示例**:
- Sales Representative (销售代表)
- Marketing Manager (市场经理) → 实际会归入Management!
- Account Executive (客户经理)
- Business Development Manager (业务拓展经理) → 归入Management!

**分类陷阱**:
- "Marketing Manager"会归入Management(因为"manager"优先)
- 纯粹的"Sales"、"Marketing Specialist"才归入此类

---

### 8. **Administrative** (行政文员类)

**关键词**:
```python
['assistant', 'secretary', 'clerk', 'administrator',
 'coordinator', 'officer']
```

**典型职位示例**:
- Executive Assistant (行政助理)
- Office Secretary (办公室秘书)
- Data Entry Clerk (数据录入员)
- Project Coordinator (项目协调员)
- Administrative Officer (行政官员)

**注意事项**:
- "Chief Executive Officer"会归入**Management**(因为"chief"优先)
- "Assistant Manager"也归入**Management**(因为"manager"优先)

---

### 9. **Technical_Operator** (技术操作类)

**关键词**:
```python
['technician', 'operator', 'driver', 'worker', 'mechanic']
```

**典型职位示例**:
- Laboratory Technician (实验室技术员)
- Machine Operator (机器操作员)
- Truck Driver (卡车司机)
- Factory Worker (工厂工人)
- Automotive Mechanic (汽车技工)

**为什么风险两极分化**:
- Manufacturing: "Machine Operator" = 流水线工人 → 易被机器人替代 (48.49%)
- Finance: "Trading System Technician" = 需要专业判断 → 难以替代 (38.60%)

---

### 10. **Other** (其他)

**定义**: 不匹配以上任何关键词的职位

**典型职位示例**:
- Barista (咖啡师)
- Chef (厨师)
- Librarian (图书管理员)
- Pilot (飞行员) → 实际会匹配到Technical_Operator!
- Social Worker (社工)

**数量占比**: 约占40% (11,956 / 30,000)

---

## 🔍 分类规则的优先级顺序

**为什么顺序很重要**:

示例职位: "**Senior Marketing Manager**"

按顺序匹配:
1. ✅ 检查是否包含 'manager' → **匹配! 归入Management**
2. ❌ (跳过后续检查)

如果顺序颠倒,可能误归入"Sales_Marketing"。

## ⚠️ 分类方法的局限性

### 局限1: **词序不敏感**

问题案例:
- "Assistant Manager" → 归入**Management** (因为"manager"优先)
- 但实际上可能更接近"Administrative"职能

解决方案:
- 可以改进为检查"assistant"出现在"manager"之前时,优先归入Administrative
- 但这会增加规则复杂度

---

### 局限2: **多重身份职位**

问题案例:
- "Technical Sales Engineer" → 归入**Engineering**
- 但实际上可能兼具Sales和Engineering职能

解决方案:
- 引入多标签分类 (一个职位可归入多个类别)
- 但会增加分析复杂度

---

### 局限3: **行业专有术语**

问题案例:
- Finance行业的"Trader" → 归入**Other**
- 但"Trader"在金融业是核心职位,应该有专门类别

解决方案:
- 可以为每个行业定制分类规则
- 但会失去跨行业可比性

---

### 局限4: **"Other"类别过大**

统计数据:
- Other类别占比: 39.8% (11,956 / 30,000)

原因:
- 许多职位名称不包含标准关键词
- 例如: "Barista", "Chef", "Plumber", "Electrician"

改进方案:
- 可以扩充关键词列表
- 或引入机器学习分类模型

### 优势4: **验证了关键发现**
- 发现Education类普遍高风险

- 这些发现具有明确的可解释性

## 📊 完整代码

```python
def classify_job_type(title):
    """
    根据职位名称中的关键词分类
    优先级: 从上到下依次匹配,首次匹配成功即返回
    """
    title_lower = str(title).lower()

    # 1. 管理类 (最高优先级)
    if any(word in title_lower for word in ['manager', 'director', 'executive', 'head', 'chief', 'president']):
        return 'Management'

    # 2. 工程技术类
    elif any(word in title_lower for word in ['engineer', 'developer', 'programmer', 'architect', 'scientist']):
        return 'Engineering'

    # 3. 分析咨询类
    elif any(word in title_lower for word in ['analyst', 'consultant', 'advisor', 'strategist']):
        return 'Analysis'

    # 4. 医疗专业类
    elif any(word in title_lower for word in ['doctor', 'physician', 'nurse', 'surgeon', 'therapist', 'dentist']):
        return 'Medical_Professional'

    # 5. 教育培训类
    elif any(word in title_lower for word in ['teacher', 'professor', 'instructor', 'educator', 'trainer']):
        return 'Education'

    # 6. 创意设计类
    elif any(word in title_lower for word in ['designer', 'artist', 'creative', 'photographer', 'writer']):
        return 'Creative'

    # 7. 销售市场类
    elif any(word in title_lower for word in ['sales', 'marketing', 'account', 'business development']):
        return 'Sales_Marketing'

    # 8. 行政文员类
    elif any(word in title_lower for word in ['assistant', 'secretary', 'clerk', 'administrator', 'coordinator', 'officer']):
        return 'Administrative'

    # 9. 技工操作类
    elif any(word in title_lower for word in ['technician', 'operator', 'driver', 'worker', 'mechanic']):
        return 'Technical_Operator'

    # 10. 其他
    else:
        return 'Other'
```
